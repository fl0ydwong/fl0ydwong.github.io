蟒语多线程三日谈

发表于 2017-11-15 | 阅读次数:
| 字数统计: 694

写程序的李查德在学习python的时候遇上了点难题。

第一日

李查德写了一个demo，分别用单线程和多线程执行10亿次整数乘法操作，他发现，多线程运行时间比单线程慢了将近一倍。这让李查德有点困惑。

“多线程的效率应该比单线程高才对啊”，李查德挠着头自言自语许久。

“因为python解释器不支持并行”

一个巨大的身形出现在李查德后方，怀里还抱着一包分享装乐事薯片，边塞着边给李查德指点江山。
他是王可达，人称达哥，技术大牛，坐李查德隔壁，常给旁边的李查德分享“人生经验”。

“达哥，python不是支持多线程么”

“你说的是并发，这个python确实支持。它指的是python解释器可以运行多个线程，但在同一时间只能有一个线程执行。也就是说，你这四核i7不管怎么折腾也只能用得上一核。”

看着李查德仍旧疑惑的眼神，王可达从桌上拽了一张纸，一支笔，大手一挥，画了幅图。

“并发呢，好比有两队人在排一个煎饼摊，谁来拿煎饼要听摊主的。并行呢，相当于两队人排两个煎饼摊，这两队人依次同步来拿煎饼。这里的两队人就是两个线程，摊主就是cpu。它的调度算法让这两个线程看上去是在同时执行的。明白了吧？”

“浅显易懂，生动形象，给达哥点赞。不过达哥，我还有个问题，你刚说python解释器不支持并行，什么原理啊？”

“听过GIL么，少年？”

“没有”

说着，王可达从桌上堆得一落书中抽了一本出来。

“这本《python核心编程》送你了，第四章好好看看。”

李查德接过书，谢过达哥后，翻了翻。

以下是王可达划线的地方。

P125页

Python 代码的执行是由 Python 􏰒􏰓􏰔虚拟机(又名解释器主􏰕环)进行控制的。Python 在 设计时是这样考虑的，在主􏰖循环中同时只能有一个控制线程在执行，就像单核 CPU 系统中的多进程一样。内存中可以有许多程序，但是在任意给定时刻只能有一个程序在运行。同理，尽管 Python 解释器中可以运行多个线程，但是在任意给定时刻只有一个线程会被解释器执行。

对 Python 􏰗􏰘虚拟机的访问是由全􏰙局解释器􏰚(GIL)控制的。这个􏰌锁就是用来保证同时只能有一个线程运行的。
