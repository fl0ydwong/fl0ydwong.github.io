之前搭建这个博客的时候，我曾在网上参考过不少牛人写的有关github和jekyll新手入门的文章。虽然它们对我搭博有很大的帮助，但又觉得它们大多都保持了牛人们极简的风格，不太适合新手。所以写这篇博文的初衷，一是作为自己对之前搭博过程的详细总结，二是希望可以写出一篇较为完整详尽的新手搭博攻略。

关于Github：




github是一种的分布式代码托管网站，在业界被广泛作为一种重要的项目开源平台以及『社交网站』。阮神有一句比喻的相当形象——『程序员的Facebook』。对于github的使用,这里我们首先关注的是由github提供的静态网页托管功能——github page。有了这个功能用户就可以实现对个人静态网站架设和托管，简单来说就是用户不需要自己架设服务器就可以把个人网站发布在网络上。

关于Jekyll

为了方便用户架设静态网站的这样需求，github推出了Jekyll这个静态博客搭建工具。jekyll中的模板可以使用HTML，MarkDown，Liquid等标记语言进行编写。Jekyll也为用户提供了一些比较简洁的模板，在搭建博客初期使用这些模板是不错的选择。

##Push到github上：

上面我们完成了使用Jekyll博客的本地搭建，下面我们需要把博客发布到github上去。

  • 创建README.md文档用于记录项目更新情况：touch README

  • 上传至队列：git add README

  • 缓存提交：git commit -m ‘提交声明’

  • 上传至服务器

  • 更新：git remote add origin http://github.com/yourname/仓库名.git

  • 上传：git push origin master

好了，现在我们使用github和jekyll搭建了一个自己的静态博客。
